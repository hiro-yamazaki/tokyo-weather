# お天気グランス（東京・6区）API版

## ① 課題名

お天気グランス（東京・6区）API版

## ② 課題内容（どんな作品か）

Open-Meteo API（APIキー不要）を使った、東京の天気を「とことん見やすく」表示するWebアプリです。忙しいときに「とりあえず天気見とこ」で、**今日と今週がパッと分かる**ことを目指しました。

- 東京23区のうち主要6区（千代田・中央・豊島・新宿・渋谷・品川）を**ボタンで切替**
- 上段に「**いま傘いる？**」の結論＋現在の気温・天気・朝/昼/夜・最高/最低・降水/湿度/風
- 「これから」は**気温と降水確率を折れ線グラフ**で表示
- 「今週」は7日分を**気温レンジバー・気温差・降水**で一覧
- **◀▶やスワイプ**で 昨日 ↔ 今日 ↔ 翌日… と日替え

## ⑤ 工夫した点・こだわった点

- いきなり全部を作らず、**まずAPIで天気を取得して表示 → 折れ線グラフ化 → 日替え** と段階的に実装しました。
- 「**数字は黒、色は意味（気温の寒暖／降水の量／曜日）**」とルールを決め、認知科学・UXを意識した配置と余白にしました。
- 「いま傘いる？」を、その日の降水確率だけでなく **「いま雨か／これから雨か」** で3分岐（はい／持っていって／いいえ）にしました。
- **APIキーが不要なAPI**を選び、GitHubに鍵を上げないよう配慮しました。

## ⑥ 難しかった点・次回トライしたいこと（又は機能）

- 折れ線グラフを（ライブラリを使わず）**自分で座標を計算して描く**ところが難しく、AIと相談しながら実装しました。
- 地名→緯度経度の検索が日本語（漢字）だとうまくいかず、**主要都市の座標を内蔵**して解決しました。
- 次回は、**現在地の自動取得**や、**雨雲レーダー（直近の雨）**、ホーム画面に追加できる対応（PWA）にトライしたいです。

## ⑦ フリー項目（感想、シェアしたいこと等なんでも）

最初は「天気を表示するだけ」でしたが、API＋グラフ＋日替えと足していくうちに、自分が毎日開きたくなるものになりました。APIを使うと、ただの数字が **「傘いる？／服どうする？」という意思決定** に変わるのが面白かったです。
