# 🌤 お天気グランス（API課題）

忙しいときに「とりあえず天気見とこ」で、**今日と今週をサッと見る**ためのアプリ。
スマホ縦画面前提・前回見た都市を記憶して、開いたら即表示。

## 使い方

1. `index.html` を **Chrome** で開く（下の「開き方」を参照）
2. 上の入力欄で都市を選ぶ／入力（主要都市はオートコンプリート候補が出る）
3. 「今日」（現在の気温・天気・体感・最高/最低・降水・今日の時間別）と「今週」7日が表示される
4. 見た都市は記憶され、次に開くと自動でその都市を表示

## 開き方（ローカル）

- **かんたん**：`index.html` を Chrome で開く（ダブルクリック→既定がSafariなら右クリックで「Chromeで開く」）
  - ※Safari はローカルファイル(`file://`)から外部APIへの通信をブロックすることがあるため Chrome 推奨
- **確実**：このフォルダで簡易サーバを立て、`http://localhost:8791/` を開く
  ```bash
  python3 -m http.server 8791
  ```

## 使用API（どちらも APIキー不要・無料）

| 用途 | API |
|---|---|
| 都市名 → 緯度・経度（※内蔵リストに無い都市のみ） | [Open-Meteo Geocoding API](https://open-meteo.com/en/docs/geocoding-api) |
| 緯度経度 → 現在 / 時間別 / 週間 の予報 | [Open-Meteo Forecast API](https://open-meteo.com/en/docs) |

> 🔒 **APIキーは未使用** ＝ GitHub に鍵が漏れる心配なし（今週の課題の注意事項に対応）。
> 主要都市（47都道府県庁所在地）の座標はアプリに内蔵し、漢字入力でも即ヒットさせている
> （Open-Meteo のジオコーディングは「東京」など漢字だとヒットしないため）。

## 講義（0613「APIを触ってみよう」）の活用

- `fetch` でデータ取得 → 返ってきた **オブジェクト/配列**から値を取り出し → `for` でループ → **jQuery `.append()`** で表示
  という講義の王道パターンをそのまま使用（時間別・週間の描画が該当）。

## コードの追い方

ロジックは **実行順の STEP 番号**でコメントしてある。`index.html STEP 01` から始まり、
`script.js` へ続く。ファイルをまたぐ所は「→ STEP◯◯」で橋渡し。部品（関数・変換表）は
「使うのは STEP◯◯」と相互参照付き。
